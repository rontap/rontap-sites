Extending JS with tools

JavaScript is awesome, but even with EMCAScript 6, it lacks many, easy-to-implement but super useful functions.
JSPlus aims to exend JS in a quiet, yet powerful way.
It provides partial replacement to the outdated and slow JQuery libraries, ex. $( css );
It adds shuffeling, mixing arrays and numbers, can generate and transpose colors and provides stats for strings. Additionally, it implements a PHP $_GET style URL reader and writer; making easier to use less and less PHP and improving compatiblity.
It provides a modern, yet cross compatible modular system. Parts of NXT.JS that have not been linked initially, can be used later, on-demand, making initial website load quicker, especially on slow netowks.



HTML elements with material design

home baked material design


NXT.JS provides simple to implement, lightweight material design elements.
The main focus is minimising the required code on the website, while maintaing a balance with understandability.
Some parts need the design-input.js file to parse them into their final form.

The API is extremely fast and modular, a typical websites loads under 250ms in a desktop connection, and under 3000ms in a poor 3G connection.

NXT.JS guide

DOM manipulation



in NXT.JS, you can acces a single element with $, mostly used for accessing id-s and in smaller websites
$$, however works like the JQuery $, returning an array of the found Nodes.
Examples can be seen below:


$("#id"); //single item selected
$("h1.sure .one").style;
$("div").innerHTML=""; //replacing innerHTML

$$("div")[5].innerHTML=""; //replacing the 5th div's innerHTML
$$("span>div:not(.on)")[i].style.classList.remove("list");
//removing a class of elements, in a for cycle

$$("code").indexOf( NodeElement );
//checking the index of NodeElement in the code NodeList


Resource-heavy websites, which update the DOM quickly, can bottleneck the usability and the framerate.
NXT.JS provides a simple kill-switch for accessing and manipulating dom


$("#id"); //single item selected
$DOMInteraction(false);
$("#id2"); //throws Error, doesnt change mode
$DOMInteraction(false); //turning interaction back on

prototype JS

Extending prototypes of arrays and strings

NXT.JS provides an extended and self-explanatory prototype library
These include:

Arrays

  • [].max() //returns the maximum in an array
  • [].min()
  • [].last()
  • [].shuffle() //returns a shuffled array
  • [].isSame() //returns boolean, whether elements are same
    •

    Math

  • Math.spread( Array ); //returns the spread of elements in an array
  • Math.sum( Array ); //returns the sum of all elements in an array
  • Math.product( Array ); //returns the product (all items multiplied together)
  • Math.avg( Array ); //returns the average
  • Math.prime( Number ); //returns all numbers that divide the given number
  • Math.isPrime( Array ); //returns boolean, whether the given number is a prime
  • Math.randInt( Number ); //random Integer up to the given number
  • Math.choose( n , k ); //permutation, returns the n-choose-k of n and k (both numbers)
    •

    Typing

    JS is a weakly typed language, however sometimes it is neccesary for strict type control. a typeof x is not satisfactory, as it can only differenciate between Objects, Primitives and special cases.
    in JSPlus, the prototype is extended with a type variable.

    var a = "hello";
    a.type; //String

    var daysSinceAccident = 0 ;
    (daysSinceAccident).type //Number

    var1 = false; //.type = Boolean
    var2 = "0"; //.type = String
    var3 = 0; //.type = Number

    PHP GET in JS

    $_GET acces and use


    Read more about PHP's $_GET parameter here .
    This feaute makes cross-site communication easier, and makes sharability better (because it is not stored in localStorage)
    This is a near 1:1 feature implementation for the $_GET variable extensively used by PHP. It is cross-compatible with that, although accessing in NXT.JS is different.

    //Example page with header: abc.html#a=5&joy=freude%20schonen%20gotterfunken&is=false
    $_GET("is"); //returns the value of element: false
    $_GET("joy"); //returns the URI decoded string "freude schonen gotterfunken"

    $_GET().argumentList(); //returns array of arguments : [ "a" , "joy" , "is"];
    $_GET().length() // returns the length of arguments. Same as $_GET().argumentList().length;

    Dependencies

    Packages, fonts and misc. used by NXT JS for its design

    Fonts

    Several fonts are used by NXT.JS to ensure an appealing look.

  • Montserrat used for titles, headers and important, short texts.
  • Open Sans used for all kind of text. all styles imported.
  • Material Icons Font used for fonts, applied using i.material-icons. Not included by default, but advised to do so.

    • Design

    NXT.JS follows the material design principles outlined here.

    If you want a one-on-one material design, use JSPlus with Material Light.
